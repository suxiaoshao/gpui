# 截图与快捷键 - 截图叠层测试

## 用例：拖拽选择截图区域

### 测试目标

验证截图叠层可通过拖拽选择区域，并把截图作为测试输入附加。

### 数据隔离

截图区域必须选择测试窗口或空白测试页面，不能截取用户真实聊天、密钥、邮件或隐私内容。

### 测试前提

- 应用已使用测试配置启动。
- 屏幕上打开一个无隐私内容的测试窗口。
- 截图叠层入口可用。

### 测试数据

- 拖拽区域：测试窗口左上角向右下角拖拽约 `400x240` 像素。
- 期望输入框文本：`请描述这张测试截图`。

### 测试步骤

1. 点击聊天表单中的截图按钮。
2. 在截图叠层出现后，从测试窗口左上角按住鼠标。
3. 向右下角拖拽约 `400x240` 像素。
4. 松开鼠标。
5. 回到聊天输入框，填写 `请描述这张测试截图`。
6. 观察输入框附件区域。

### 预期结果

- 截图叠层显示选择框。
- 松开鼠标后截图附件出现在聊天输入框附近。
- 附件预览不包含用户真实敏感内容。
- 输入框保留 `请描述这张测试截图`。

### 边缘情况

- 拖拽区域过小，例如小于 `5x5` 像素时，应取消截图或显示无效区域提示，不能生成空白附件。

### 清理

删除截图附件，清空输入框，关闭应用并删除测试数据目录。

## 用例：按 Escape 取消截图

### 测试目标

验证截图叠层可被 `Escape` 取消，取消后不添加附件。

### 数据隔离

不截取任何用户真实内容。

### 测试前提

- 聊天输入框为空。
- 截图按钮可点击。

### 测试数据

- 取消键：`Escape`。

### 测试步骤

1. 点击聊天表单中的截图按钮。
2. 确认截图叠层出现。
3. 按下 `Escape`。
4. 观察聊天输入框附件区域。

### 预期结果

- 截图叠层关闭。
- 聊天输入框没有新增截图附件。
- 原窗口焦点恢复到截图前的应用窗口。

### 边缘情况

- 截图叠层取消后再次点击截图按钮，应能重新进入截图状态。

### 清理

关闭应用并删除测试配置。
