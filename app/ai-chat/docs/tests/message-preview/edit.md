# 预览 Message - 编辑测试

## 用例：编辑助手消息并保存

### 测试目标

验证预览窗口能保存对单条消息内容的修改。

### 数据隔离

只编辑测试消息。测试前用命令行预置消息，测试后删除测试数据库。

### 测试前提

- 会话 `QA 命名助手` 中存在助手消息 `QA 助手响应内容`。
- 当前没有正在生成的回复。

### 测试数据

- 原内容：`QA 助手响应内容`。
- 新内容：`QA 助手响应内容 - 已编辑`。

### 测试步骤

1. 使用测试数据目录启动应用。
2. 在侧边栏点击 `QA 命名助手`。
3. 点击 `QA 助手响应内容` 消息旁边的预览按钮。
4. 点击预览窗口中的编辑按钮。
5. 在内容编辑框中全选原内容。
6. 填写 `QA 助手响应内容 - 已编辑`。
7. 点击保存按钮。
8. 回到主页消息列表。

### 预期结果

- 保存后预览窗口显示 `QA 助手响应内容 - 已编辑`。
- 主页消息列表显示 `QA 助手响应内容 - 已编辑`。
- 原内容 `QA 助手响应内容` 不再作为该消息正文显示。

### 边缘情况

- 如果保存过程中出现错误，对话框不应关闭，并应显示错误提示；主页消息仍保持原内容。

### 清理

关闭应用并删除测试数据库。

## 用例：编辑为空内容时阻止保存

### 测试目标

验证消息内容为空时保存被阻止。

### 数据隔离

只使用测试消息，不修改真实会话。

### 测试前提

- 已打开 `QA 助手响应内容` 的预览窗口。
- 已进入编辑状态。

### 测试数据

- 非法内容：空字符串。

### 测试步骤

1. 在内容编辑框中全选所有文本。
2. 按删除键清空内容。
3. 观察保存按钮状态。
4. 点击保存按钮，如果按钮仍可点击。

### 预期结果

- 保存按钮应禁用，或点击后显示内容不能为空的错误。
- 预览窗口不关闭。
- 原消息内容仍为 `QA 助手响应内容`。

### 边缘情况

- 内容只包含空格或换行时，应按空内容处理，不能保存为空白消息。

### 清理

点击取消按钮或关闭预览窗口，关闭应用并删除测试数据库。
