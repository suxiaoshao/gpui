# 预览 Message - 预览测试

## 用例：从主页打开助手消息预览

### 测试目标

验证预览窗口显示被选中的助手消息内容。

### 数据隔离

测试前用命令行在测试数据库中预置消息，不能打开用户真实消息。

### 测试前提

- 测试数据库包含会话 `QA 命名助手`。
- 会话中存在助手消息 `QA 助手响应内容`。

### 测试数据

- 目标会话：`QA 命名助手`。
- 目标消息：`QA 助手响应内容`。

### 测试步骤

1. 使用测试数据目录启动应用。
2. 在侧边栏点击 `QA 命名助手`。
3. 在消息列表中找到 `QA 助手响应内容`。
4. 点击该消息旁边的预览按钮。
5. 观察预览窗口标题和正文区域。

### 预期结果

- 预览窗口打开。
- 正文区域显示 `QA 助手响应内容`。
- 预览窗口不会显示其他会话的消息。

### 边缘情况

- 如果同一会话里存在两条相同开头的消息，预览窗口应显示完整目标消息，而不是按前缀误匹配。

### 清理

关闭预览窗口，关闭应用并删除测试数据库。

## 用例：关闭预览窗口不改变消息

### 测试目标

验证只预览不编辑时，关闭窗口不会改写消息。

### 数据隔离

使用测试数据库，关闭窗口后通过测试数据检查消息仍存在。

### 测试前提

- 已打开 `QA 助手响应内容` 的预览窗口。

### 测试数据

- 原始消息内容：`QA 助手响应内容`。

### 测试步骤

1. 在预览窗口中不要点击编辑按钮。
2. 点击预览窗口关闭按钮。
3. 回到主页消息列表。
4. 找到原助手消息。

### 预期结果

- 预览窗口关闭。
- 主页消息列表仍显示 `QA 助手响应内容`。
- 消息没有新增、删除或变更。

### 边缘情况

- 按 `Escape` 关闭预览窗口时，结果应与点击关闭按钮一致。

### 清理

关闭应用并删除测试数据库。
