# 预览 Message - 元数据与内容展示测试

## 用例：展示消息角色和会话来源

### 测试目标

验证消息预览展示角色、会话来源和正文内容。

### 数据隔离

通过测试数据库预置消息元数据，不读取用户真实消息。

### 测试前提

- `QA 命名助手` 中存在 assistant 角色消息 `QA 助手响应内容`。

### 测试数据

- 会话标题：`QA 命名助手`。
- 消息角色：`assistant`。
- 消息内容：`QA 助手响应内容`。

### 测试步骤

1. 使用测试数据目录启动应用。
2. 打开 `QA 命名助手`。
3. 点击 `QA 助手响应内容` 的预览按钮。
4. 查看预览窗口的角色字段。
5. 查看预览窗口的会话来源字段。
6. 查看正文区域。

### 预期结果

- 角色字段显示 `assistant` 或对应的助手角色文案。
- 会话来源显示 `QA 命名助手`。
- 正文区域显示 `QA 助手响应内容`。

### 边缘情况

- 对 developer 角色消息执行同样操作时，角色字段应显示 developer 或对应开发者角色文案，不能误显示为 user。

### 清理

关闭预览窗口，关闭应用并删除测试数据库。

## 用例：展示长文本消息

### 测试目标

验证长文本消息在预览窗口中可滚动查看，不遮挡操作按钮。

### 数据隔离

长文本消息通过命令行写入测试数据库，不使用用户真实长消息。

### 测试前提

- `QA 长文本会话` 中存在长消息，内容以 `QA 长文本开始` 开头，以 `QA 长文本结束` 结尾。

### 测试数据

- 会话：`QA 长文本会话`。
- 长文本开头：`QA 长文本开始`。
- 长文本结尾：`QA 长文本结束`。

### 测试步骤

1. 使用测试数据目录启动应用。
2. 打开 `QA 长文本会话`。
3. 点击长文本消息旁边的预览按钮。
4. 确认预览正文顶部显示 `QA 长文本开始`。
5. 滚动预览正文到底部。
6. 确认底部显示 `QA 长文本结束`。
7. 观察编辑、删除、关闭等操作按钮位置。

### 预期结果

- 长文本可在预览窗口内滚动查看。
- 顶部和底部内容均可到达。
- 操作按钮不被正文遮挡。

### 边缘情况

- 缩小窗口后，正文区域应继续可滚动，按钮仍可点击。

### 清理

关闭预览窗口，关闭应用并删除测试数据库。
