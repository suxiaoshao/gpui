# ai-chat 测试步骤文档

本文档与 `docs/features/` 一一对应。测试目标是验证用户可见行为，不验证 Rust 模块结构。每条测试必须写清具体输入、按钮、观察点和清理方式，方便人工执行，也方便 agent 使用 `computer-use` 执行。

## 数据隔离与预置规则

- 不使用用户真实配置、真实会话、真实 API Key、真实导出目录作为测试对象。
- 每次手测使用独立测试数据目录或测试数据库。推荐将应用数据目录指向临时目录，例如 `/tmp/ai-chat-qa-data`。
- 如果测试目标不是“新建流程”，但需要文件夹、会话、消息、模板或快捷键作为前置数据，优先在启动 app 前用命令行写入测试数据库。
- 测试结束后删除测试数据库、导出文件和临时配置。不要清理用户真实数据目录。
- 任何截图、日志、issue 评论和 PR 描述都不得包含真实 API Key、真实聊天内容或用户隐私数据。

## 命令行预置数据建议

具体命令应以当前数据库 schema 为准。预置流程建议如下：

1. 在临时目录初始化 app 数据，例如 `/tmp/ai-chat-qa-data`。
2. 运行 app 一次，让迁移创建 SQLite 数据库。
3. 关闭 app。
4. 使用 `sqlite3 <测试数据库路径>` 写入测试文件夹、会话、消息、模板和快捷键。
5. 启动 app，并确认页面只显示 `QA` 前缀的数据。

预置数据不要写入默认用户数据目录。若不能确认数据库路径，停止测试并先定位测试环境，而不是继续操作真实数据。

## 用例格式要求

每个非 README 测试文档必须包含以下结构：

- `测试目标`
- `数据隔离`
- `测试前提`
- `测试数据`
- `测试步骤`
- `预期结果`
- `边缘情况`
- `清理`

每个步骤必须能直接执行。不要写“创建一个会话”或“填写信息”；要写“在名称输入框填写 `QA 命名助手`，在图标输入框填写 `🧩`，点击 `创建` 按钮”。
