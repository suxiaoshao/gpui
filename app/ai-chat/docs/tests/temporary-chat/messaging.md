# 临时对话 - 发送测试

## 用例：发送临时消息

### 测试目标

验证临时对话能发送测试消息，并且不会自动保存为普通会话。

### 数据隔离

使用测试 provider 或 mock provider，不能使用真实 API Key 或真实付费模型。

### 测试前提

- 临时对话窗口已打开。
- 测试模型 `qa-model-small` 可用或 mock provider 可返回固定响应。

### 测试数据

- 输入内容：`请用一句话回复临时 QA 消息`。
- 预期 mock 响应：`临时 QA 响应内容`。

### 测试步骤

1. 点击临时对话输入框。
2. 填写 `请用一句话回复临时 QA 消息`。
3. 点击发送按钮。
4. 等待生成结束或 mock 响应返回。
5. 观察临时对话消息列表。
6. 打开主窗口侧边栏。

### 预期结果

- 临时对话中显示用户消息 `请用一句话回复临时 QA 消息`。
- 临时对话中显示助手响应 `临时 QA 响应内容` 或明确的测试错误提示。
- 主窗口侧边栏不自动新增普通会话。

### 边缘情况

- 如果测试模型不可用，应显示模型不可用错误，输入内容不应丢失。

### 清理

点击临时对话清空按钮并确认，关闭临时对话窗口，删除测试配置。

## 用例：空输入不能发送

### 测试目标

验证临时对话空输入或空白输入不会发送。

### 数据隔离

不涉及持久数据；仍使用测试配置运行。

### 测试前提

- 临时对话窗口已打开。
- 输入框为空。

### 测试数据

- 空白输入：三个空格。

### 测试步骤

1. 点击临时对话输入框。
2. 填写三个空格。
3. 观察发送按钮状态。
4. 如果发送按钮可点击，点击发送按钮。

### 预期结果

- 发送按钮应禁用，或点击后显示输入不能为空。
- 消息列表不新增用户消息。
- 不触发模型请求。

### 边缘情况

- 输入只包含换行时，应按空输入处理。

### 清理

清空输入框，关闭临时对话窗口。
