# 临时对话 - 入口测试

## 用例：通过菜单打开临时对话

### 测试目标

验证临时对话可从应用菜单打开。

### 数据隔离

使用测试数据目录，打开临时对话不应读取用户真实会话。

### 测试前提

- 应用已启动。
- 临时对话窗口未打开。

### 测试数据

- 菜单路径：`AI Chat` > `打开临时对话`。

### 测试步骤

1. 在系统菜单栏点击 `AI Chat` 应用菜单。
2. 点击 `打开临时对话` 菜单项。
3. 观察前台窗口。

### 预期结果

- 临时对话窗口打开。
- 窗口中显示聊天输入框。
- 侧边栏不会新增普通会话。

### 边缘情况

- 临时对话已打开时再次执行菜单操作，应聚焦已有临时对话窗口。

### 清理

关闭临时对话窗口。

## 用例：通过快捷键打开临时对话

### 测试目标

验证临时对话快捷键能打开临时窗口。

### 数据隔离

快捷键使用测试配置，不修改用户真实全局快捷键。

### 测试前提

- 测试配置中临时对话快捷键为 `Cmd+Shift+9`。
- 临时对话窗口未打开。

### 测试数据

- 快捷键：`Cmd+Shift+9`。

### 测试步骤

1. 聚焦主窗口或桌面。
2. 按下 `Cmd+Shift+9`。
3. 观察前台窗口。

### 预期结果

- 临时对话窗口打开并获得焦点。
- 输入框可直接输入文本。

### 边缘情况

- 如果快捷键注册失败，应在设置或通知中显示明确错误，不能静默失效。

### 清理

关闭临时对话窗口，删除测试快捷键配置。
