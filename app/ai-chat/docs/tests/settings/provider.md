# 设置 - 提供方测试

## API Key 默认掩码

### 测试目标

验证敏感字段默认不明文显示，且不会在切页后保持暴露状态。

### 数据隔离

只在测试配置中使用占位 API Key `sk-test-redacted-value`。不得输入、截图或记录真实 API Key。

### 测试前提

- app 使用测试数据目录启动。
- 设置窗口已打开并进入“提供方”页。

### 测试数据

- OpenAI API Key：`sk-test-redacted-value`

### 测试步骤

1. 找到 OpenAI 的 “API Key” 输入框。
2. 在 “API Key” 输入框填写 `sk-test-redacted-value`。
3. 点击输入框外部或触发保存。
4. 关闭设置窗口。
5. 重新打开“设置 > 提供方”。
6. 观察 “API Key” 输入框。
7. 点击“显示 API Key”按钮。
8. 点击“隐藏 API Key”按钮。
9. 切换到“通用”页，再切回“提供方”页。

### 预期结果

- 第 5 步后 API Key 默认以掩码显示。
- 第 7 步后才显示 `sk-test-redacted-value`。
- 第 8 步后重新掩码。
- 第 9 步返回后不应意外明文显示。

### 边缘情况

- 任何通知、日志和截图都不应包含真实密钥。若执行 agent 需要截图，必须先确认字段处于掩码状态。

### 清理

在测试配置中清空 OpenAI API Key，或删除测试数据目录。

## 编辑 Base URL 和 HTTP Proxy

### 测试目标

验证提供方文本配置可保存，长文本不遮挡 UI。

### 数据隔离

只修改测试配置，不修改用户真实提供方配置。

### 测试前提

- 设置窗口已打开并进入“提供方”页。

### 测试数据

- Base URL：`https://example.test/v1`
- HTTP Proxy：`http://127.0.0.1:18080`

### 测试步骤

1. 找到 OpenAI 的 “Base URL” 输入框。
2. 选中原内容并填写 `https://example.test/v1`。
3. 找到 OpenAI 的 “HTTP Proxy” 输入框。
4. 选中原内容并填写 `http://127.0.0.1:18080`。
5. 切换到“通用”页。
6. 再切回“提供方”页。

### 预期结果

- Base URL 仍显示 `https://example.test/v1`。
- HTTP Proxy 仍显示 `http://127.0.0.1:18080`。
- 两个输入框内容可读，不溢出到相邻控件。

### 边缘情况

- 清空 HTTP Proxy 后应保存为空值，不应自动恢复旧测试值。

### 清理

清空测试配置中的 Base URL 和 HTTP Proxy，或删除测试数据目录。

## 空 API Key 的未配置状态

### 测试目标

验证 OpenAI API Key 为空时，产品不会把 OpenAI 当作可用提供方静默发送。

### 数据隔离

只在测试配置中清空 API Key。不要修改用户真实配置。

### 测试前提

- 测试数据库中有会话 `QA 命名助手`。
- OpenAI API Key 当前为测试占位值或空。

### 测试数据

- 会话：`QA 命名助手`
- 消息：`请用一句话回复 QA 测试消息`

### 测试步骤

1. 打开“设置 > 提供方”。
2. 清空 OpenAI 的 “API Key” 输入框。
3. 回到主窗口。
4. 打开 `QA 命名助手`。
5. 在模型选择中选择 OpenAI 模型，或观察 OpenAI 是否显示为不可用。
6. 在聊天表单输入 `请用一句话回复 QA 测试消息`。
7. 点击“发送”按钮。

### 预期结果

- OpenAI 未配置状态应可见。
- 不应静默发送失败。
- 若允许点击发送，应在当前消息或通知中显示需要配置 API Key 的错误。

### 边缘情况

- 空 API Key 不应被保存为有效密钥。

### 清理

删除测试数据目录，或恢复测试 API Key 占位值。
