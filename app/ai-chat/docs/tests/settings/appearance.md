# 设置 - 外观测试

## 用例：切换测试主题并保存

### 测试目标

验证外观主题设置保存后在测试配置中生效。

### 数据隔离

使用测试配置目录，不修改用户真实主题偏好。

### 测试前提

- 应用已使用测试配置目录启动。
- 设置窗口可打开。

### 测试数据

- 主题选择：`浅色`。

### 测试步骤

1. 打开设置窗口。
2. 点击左侧导航中的 `外观`。
3. 在主题选择控件中点击 `浅色`。
4. 点击保存按钮。
5. 关闭设置窗口。
6. 观察主窗口背景、文字和边框。

### 预期结果

- 主窗口切换到浅色外观。
- 文字和图标仍有足够对比度。
- 只写入测试配置目录。

### 边缘情况

- 切换主题后已打开的会话标签和消息列表不应出现文字与背景重叠或不可读。

### 清理

恢复测试默认主题，关闭应用并删除测试配置目录。

## 用例：外观设置取消不保存

### 测试目标

验证未保存的外观修改不会持久化。

### 数据隔离

只使用测试配置目录。

### 测试前提

- 当前测试主题为 `浅色`。
- 设置窗口已打开并停留在 `外观`。

### 测试数据

- 临时主题选择：`深色`。

### 测试步骤

1. 在主题选择控件中点击 `深色`。
2. 点击取消按钮或关闭设置窗口。
3. 重新打开设置窗口。
4. 点击左侧导航中的 `外观`。
5. 观察当前主题选择。

### 预期结果

- 当前主题仍显示为 `浅色`。
- 未保存的 `深色` 选择不应持久化。
- 主窗口外观与保存前一致。

### 边缘情况

- 如果外观切换提供即时预览，取消后应回滚到保存前主题。

### 清理

关闭应用并删除测试配置目录。
