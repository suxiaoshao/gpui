# 设置 - 通用测试

## 用例：修改测试语言设置并保存

### 测试目标

验证通用设置保存后只影响测试配置。

### 数据隔离

使用隔离测试配置目录，不能修改用户真实语言和启动偏好。

### 测试前提

- 应用已使用测试配置目录启动。
- 设置窗口未打开。

### 测试数据

- 语言选择：`简体中文`。

### 测试步骤

1. 打开设置窗口。
2. 点击左侧导航中的 `通用`。
3. 在语言选择控件中选择 `简体中文`。
4. 点击保存按钮。
5. 关闭设置窗口。
6. 重新打开设置窗口并进入 `通用`。

### 预期结果

- 保存后设置窗口仍可正常显示中文文案。
- 第 6 步语言选择显示为 `简体中文`。
- 用户真实配置目录中的语言设置不变。

### 边缘情况

- 如果点击取消按钮或关闭窗口而不保存，重新打开后应保持保存前的语言设置。

### 清理

关闭应用并删除测试配置目录。

## 用例：漏填必填通用设置时阻止保存

### 测试目标

验证通用设置中的必填字段为空时不会保存非法配置。

### 数据隔离

只修改测试配置目录。

### 测试前提

- 通用设置页面存在必填文本字段。
- 当前字段已有有效测试值。

### 测试数据

- 非法值：空字符串。

### 测试步骤

1. 打开设置窗口。
2. 点击左侧导航中的 `通用`。
3. 在必填文本字段中全选已有内容。
4. 按删除键清空字段。
5. 观察保存按钮和字段错误提示。
6. 如果保存按钮仍可点击，点击保存按钮。

### 预期结果

- 保存按钮禁用，或点击后显示必填错误。
- 设置窗口不关闭。
- 原有效配置不被空值覆盖。

### 边缘情况

- 字段只包含空格时，应按未填写处理。

### 清理

恢复原有效测试值，关闭应用并删除测试配置目录。
