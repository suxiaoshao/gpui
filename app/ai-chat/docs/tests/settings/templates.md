# 设置 - 模板测试

## 搜索模板

### 测试目标

验证模板可按名称、描述、拼音和首字母搜索。

### 数据隔离

用命令行在测试数据库中预置模板，不使用用户真实模板。

### 测试前提

- 设置窗口已打开。
- 已进入“模板”设置页。

### 测试数据

- 模板 A：名称 `QA 单提示词模板`，描述 `QA 文档测试模板`
- 模板 B：名称 `命名助手`，描述 `生成更好的名字`

### 测试步骤

1. 点击“搜索模板”输入框。
2. 输入 `QA 单提示词`。
3. 观察模板列表。
4. 选中搜索框内容并输入 `生成更好的名字`。
5. 观察模板列表。
6. 选中搜索框内容并输入 `mingming`。
7. 观察模板列表。
8. 选中搜索框内容并输入 `mmzs`。
9. 观察模板列表。
10. 清空搜索框。

### 预期结果

- 第 2 步后只显示或突出 `QA 单提示词模板`。
- 第 4 步后显示 `命名助手`。
- 第 6 和第 8 步均能匹配 `命名助手`。
- 清空后恢复完整模板列表。

### 边缘情况

- 输入不存在的 `QA 不存在模板` 时，显示空状态，不保留旧结果。

### 清理

删除测试数据库中的模板 A 和模板 B。

## 新增单 prompt 模板

### 测试目标

验证用户可以新增包含一个 prompt 的模板。

### 数据隔离

只在测试数据库中新增模板。

### 测试前提

- 设置窗口已打开。
- 已进入“模板”设置页。

### 测试数据

- 名称：`QA 单提示词模板`
- 图标：`🧪`
- 描述：`QA 文档测试模板`
- 角色：`User`
- Prompt：`请为产品功能生成命名建议。`

### 测试步骤

1. 点击“添加”或“添加模板”按钮。
2. 在“名称”输入框填写 `QA 单提示词模板`。
3. 在“图标”输入框填写 `🧪`。
4. 在“描述”输入框填写 `QA 文档测试模板`。
5. 在第 1 个 prompt 行的“角色”下拉框选择 `User`。
6. 在第 1 个 prompt 行的“提示词”输入框填写 `请为产品功能生成命名建议。`。
7. 点击“保存”或“创建”按钮。

### 预期结果

- 对话框关闭。
- 模板列表出现 `🧪 QA 单提示词模板`。
- 该模板显示 prompt 数量为 1。

### 边缘情况

- 保存后再次打开详情，角色和 prompt 内容应与测试数据一致。

### 清理

删除 `QA 单提示词模板`。

## 新增多 prompt 模板

### 测试目标

验证用户可以新增包含 Developer 和 User prompt 的模板。

### 数据隔离

只在测试数据库中新增模板。

### 测试前提

- 设置窗口已打开并进入“模板”页。

### 测试数据

- 名称：`QA 多提示词模板`
- 图标：`🧪`
- 描述：`QA 文档测试模板`
- Developer prompt：`你是一个命名助手，只输出三个中文名称。`
- User prompt：`请为产品功能生成命名建议。`

### 测试步骤

1. 点击“添加模板”按钮。
2. 在“名称”输入框填写 `QA 多提示词模板`。
3. 在“图标”输入框填写 `🧪`。
4. 在“描述”输入框填写 `QA 文档测试模板`。
5. 在第 1 个 prompt 行的“角色”下拉框选择 `Developer`。
6. 在第 1 个 prompt 行的“提示词”输入框填写 `你是一个命名助手，只输出三个中文名称。`。
7. 点击“添加提示词”按钮。
8. 在第 2 个 prompt 行的“角色”下拉框选择 `User`。
9. 在第 2 个 prompt 行的“提示词”输入框填写 `请为产品功能生成命名建议。`。
10. 点击“保存”或“创建”按钮。

### 预期结果

- 模板列表出现 `QA 多提示词模板`。
- Prompt 数量显示为 2。
- 查看详情时 prompt 顺序为 Developer 在前、User 在后。

### 边缘情况

- 第二个 prompt 漏填内容时不能保存。

### 清理

删除 `QA 多提示词模板`。

## 编辑模板

### 测试目标

验证模板元数据和 prompt 内容可编辑。

### 数据隔离

用命令行预置 `QA 单提示词模板`。

### 测试前提

- 模板列表中存在 `QA 单提示词模板`。

### 测试数据

- 新名称：`QA 单提示词模板 - 已编辑`
- 新图标：`💡`
- 新描述：`QA 编辑后的模板描述`
- 新 prompt：`请输出五个简短中文名称。`

### 测试步骤

1. 在模板列表找到 `QA 单提示词模板`。
2. 点击该行的“编辑”按钮。
3. 在“名称”输入框填写 `QA 单提示词模板 - 已编辑`。
4. 在“图标”输入框填写 `💡`。
5. 在“描述”输入框填写 `QA 编辑后的模板描述`。
6. 在第 1 个 prompt 的“提示词”输入框填写 `请输出五个简短中文名称。`。
7. 点击“保存”按钮。

### 预期结果

- 列表显示 `💡 QA 单提示词模板 - 已编辑`。
- 查看详情时描述和 prompt 内容均为新值。

### 边缘情况

- 编辑时漏填名称或图标，点击保存后对话框不应关闭，原模板不应被覆盖。

### 清理

删除编辑后的测试模板。

## 漏填名称和图标

### 测试目标

验证模板表单必填字段校验。

### 数据隔离

只使用测试数据库，且不保存无效模板。

### 测试前提

- 设置窗口已打开并进入“模板”页。

### 测试数据

- 名称：`QA 漏填校验模板`
- 图标：`🧪`

### 测试步骤

1. 点击“添加模板”按钮。
2. 保持“名称”输入框为空。
3. 在“图标”输入框填写 `🧪`。
4. 点击“保存”或“创建”按钮。
5. 观察提示。
6. 在“名称”输入框填写 `QA 漏填校验模板`。
7. 清空“图标”输入框。
8. 点击“保存”或“创建”按钮。

### 预期结果

- 第 4 步后模板不会创建，对话框不关闭。
- 第 8 步后模板不会创建，对话框不关闭。
- 用户能看到名称或图标必填提示，或保存按钮保持不可用。

### 边缘情况

- 列表中不应出现 `QA 漏填校验模板`。

### 清理

如果误创建了测试模板，删除它。

## 删除模板取消和确认

### 测试目标

验证模板删除的取消和确认路径。

### 数据隔离

用命令行预置 `QA 单提示词模板`。

### 测试前提

- 模板列表中存在 `QA 单提示词模板`。

### 测试数据

- 待删除模板：`QA 单提示词模板`

### 测试步骤

1. 在模板列表找到 `QA 单提示词模板`。
2. 点击该行的“删除”按钮。
3. 在确认对话框点击“取消”。
4. 确认列表仍显示 `QA 单提示词模板`。
5. 再次点击该行的“删除”按钮。
6. 在确认对话框点击“删除模板”或“删除”按钮。

### 预期结果

- 取消后模板仍存在。
- 确认后模板从列表移除。
- 删除后搜索 `QA 单提示词模板` 不再命中。

### 边缘情况

- 删除确认文案应明确这是破坏性操作。

### 清理

确认测试模板已删除。
