# 应用外壳 - 窗口与关于测试

## 用例：从菜单打开关于窗口

### 测试目标

验证关于窗口可从菜单打开，并展示可观察的应用信息。

### 数据隔离

关于窗口不依赖用户数据；测试仍使用隔离数据目录启动应用。

### 测试前提

- 应用已启动。
- 关于窗口未打开。

### 测试数据

- 菜单路径：`AI Chat` > `关于 AI Chat`。

### 测试步骤

1. 在系统菜单栏点击 `AI Chat` 应用菜单。
2. 点击 `关于 AI Chat` 菜单项。
3. 观察弹出的关于窗口或关于对话框。
4. 点击关于窗口的关闭按钮。

### 预期结果

- 关于窗口打开。
- 窗口中显示应用名称 `AI Chat`。
- 关闭按钮可关闭关于窗口。
- 不改变任何会话、模板、快捷键或提供方配置。

### 边缘情况

- 连续两次点击 `关于 AI Chat` 时，应聚焦已有关于窗口，或只保留一个关于对话框。

### 清理

关闭关于窗口。

## 用例：主窗口关闭后重新打开

### 测试目标

验证主窗口关闭后可通过菜单重新打开。

### 数据隔离

使用测试数据库；测试过程中不打开用户真实对话。

### 测试前提

- 应用已启动。
- 主窗口处于前台。

### 测试数据

- 菜单路径：`AI Chat` > `打开主窗口`。

### 测试步骤

1. 点击主窗口左上角关闭按钮。
2. 确认屏幕上不再显示主窗口。
3. 在系统菜单栏点击 `AI Chat` 应用菜单。
4. 点击 `打开主窗口` 菜单项。
5. 观察重新打开的窗口。

### 预期结果

- 第 1 步后主窗口关闭，但应用进程保持运行。
- 第 4 步后主窗口重新显示主页。
- 重新打开窗口不会新增测试会话或改变当前选中会话。

### 边缘情况

- 如果关闭主窗口时存在未保存设置对话框，应先取消或保存该对话框，避免将窗口生命周期测试和设置保存测试混在一起。

### 清理

关闭主窗口，删除本用例使用的临时测试数据目录。
