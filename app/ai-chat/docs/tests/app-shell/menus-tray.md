# 应用外壳 - 菜单栏与托盘测试

## 用例：菜单栏打开主窗口

### 测试目标

验证菜单栏入口能打开或聚焦主窗口。

### 数据隔离

使用测试数据目录启动应用，不读取用户真实会话和设置。

### 测试前提

- 应用已以测试数据目录启动。
- 主窗口已关闭或位于后台。

### 测试数据

- 目标菜单项：`AI Chat` > `打开主窗口`。

### 测试步骤

1. 在系统菜单栏点击 `AI Chat` 应用菜单。
2. 点击 `打开主窗口` 菜单项。
3. 观察屏幕中当前前台窗口。

### 预期结果

- 主窗口出现在前台。
- 主窗口标题或主要内容区域显示 `AI Chat` 的主页内容。
- 不创建、修改或删除任何测试会话。

### 边缘情况

- 如果主窗口已经打开，点击 `打开主窗口` 后应聚焦已有窗口，而不是打开重复主窗口。

### 清理

关闭主窗口，保留测试数据目录用于后续菜单测试。

## 用例：菜单栏打开设置窗口

### 测试目标

验证菜单栏设置入口能打开设置窗口。

### 数据隔离

只使用测试配置，不能读取或展示用户真实 API Key。

### 测试前提

- 应用已以测试数据目录启动。
- 设置窗口未打开。

### 测试数据

- 目标菜单项：`AI Chat` > `设置`。

### 测试步骤

1. 在系统菜单栏点击 `AI Chat` 应用菜单。
2. 点击 `设置` 菜单项。
3. 在设置窗口左侧导航中确认存在 `通用`、`外观`、`提供方`、`模板`、`快捷键`。

### 预期结果

- 设置窗口打开并显示设置导航。
- 提供方页面不直接展示任何真实密钥明文。

### 边缘情况

- 重复点击 `设置` 菜单项时，应聚焦已有设置窗口，不应产生多个重复设置窗口。

### 清理

关闭设置窗口。

## 用例：托盘菜单打开临时对话

### 测试目标

验证托盘入口能打开临时对话窗口。

### 数据隔离

临时对话只使用测试数据目录，不保存为用户真实会话。

### 测试前提

- 当前平台显示应用托盘图标。
- 临时对话窗口未打开。

### 测试数据

- 托盘菜单项：`打开临时对话`。

### 测试步骤

1. 点击系统托盘中的 `AI Chat` 图标。
2. 点击 `打开临时对话` 菜单项。
3. 观察新窗口中的输入框和发送按钮。

### 预期结果

- 临时对话窗口打开。
- 窗口中显示消息输入框和发送按钮。
- 不自动创建普通会话。

### 边缘情况

- 如果临时对话已打开，再次点击托盘菜单项应聚焦已有临时对话窗口。

### 清理

关闭临时对话窗口；如果临时窗口中产生测试消息，清空临时会话。
