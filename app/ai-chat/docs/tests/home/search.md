# 主页 - 会话搜索测试

## 用例：按中文标题搜索并打开会话

### 测试目标

验证主页搜索能按会话标题命中并打开指定会话。

### 数据隔离

不要使用用户真实会话。测试前用命令行向测试数据库预置会话 `QA 命名助手`。

### 测试前提

- 应用尚未启动。
- 测试数据库已预置会话 `QA 命名助手`，消息内容为 `请用一句话回复 QA 测试消息`。

### 测试数据

- 搜索词：`命名助手`。
- 目标会话：`QA 命名助手`。

### 测试步骤

1. 使用测试数据目录启动应用。
2. 在主页侧边栏点击搜索按钮。
3. 在搜索输入框填写 `命名助手`。
4. 点击搜索结果中的 `QA 命名助手`。
5. 观察主页当前打开的标签页标题。

### 预期结果

- 搜索结果列表显示 `QA 命名助手`。
- 点击结果后打开 `QA 命名助手` 会话。
- 当前标签页标题显示 `QA 命名助手`。

### 边缘情况

- 搜索词前后添加空格 ` 命名助手 ` 时，应仍能命中同一会话或明确按当前搜索规则展示无结果，不能打开错误会话。

### 清理

关闭应用并删除测试数据库。

## 用例：按无结果关键词搜索

### 测试目标

验证搜索无结果时显示空状态且不改变当前会话。

### 数据隔离

测试数据库只包含 `QA 命名助手`，不使用用户真实会话。

### 测试前提

- 应用已使用测试数据库启动。
- 当前打开会话为 `QA 命名助手`。

### 测试数据

- 搜索词：`qa-not-existing-conversation`。

### 测试步骤

1. 在主页侧边栏点击搜索按钮。
2. 在搜索输入框全选已有内容并填写 `qa-not-existing-conversation`。
3. 观察搜索结果区域。
4. 按下 `Escape`。
5. 观察当前会话标题。

### 预期结果

- 搜索结果区域显示无结果状态。
- 不自动创建新会话。
- 按 `Escape` 后搜索面板关闭。
- 当前会话仍为 `QA 命名助手`。

### 边缘情况

- 清空搜索输入框后，应显示默认搜索状态或最近结果，不应残留 `qa-not-existing-conversation` 的无结果提示。

### 清理

关闭应用并删除测试数据库。

## 用例：搜索快捷键只在主页生效

### 测试目标

验证主页搜索快捷键不会在设置窗口中误触发主页搜索。

### 数据隔离

使用测试配置，不读取用户自定义快捷键。

### 测试前提

- 应用已使用测试数据目录启动。
- 主页窗口和设置窗口都可打开。

### 测试数据

- 搜索快捷键：应用默认搜索快捷键。

### 测试步骤

1. 聚焦主页窗口。
2. 按下应用默认搜索快捷键。
3. 确认主页搜索输入框出现。
4. 按下 `Escape` 关闭搜索。
5. 打开设置窗口并聚焦设置窗口。
6. 再次按下应用默认搜索快捷键。

### 预期结果

- 第 2 步在主页打开搜索输入框。
- 第 6 步不应在设置窗口上方打开主页搜索面板。
- 设置窗口当前页面保持不变。

### 边缘情况

- 如果设置窗口本身有搜索功能，应只触发设置窗口自己的搜索入口，不应切换或污染主页状态。

### 清理

关闭设置窗口和应用，删除测试数据目录。
