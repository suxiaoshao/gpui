# 主页 - 对话 - 消息发送测试

## 正常发送消息

### 测试目标

验证用户在普通会话中选择模型并发送消息后，消息列表显示用户消息和助手响应。

### 数据隔离

用命令行预置会话 `QA 命名助手`。如果要真实调用模型，只使用测试账号和非敏感内容。

### 测试前提

- app 使用测试数据目录启动。
- `QA 命名助手` 已存在。
- 测试模型 `qa-model-small` 可用，或 mock provider 已配置为返回固定响应。

### 测试数据

- 会话：`QA 命名助手`
- 消息：`请用一句话回复 QA 测试消息`

### 测试步骤

1. 在侧边栏点击 `QA 命名助手`。
2. 点击聊天表单中的模型选择控件。
3. 在模型列表中点击 `qa-model-small`。
4. 点击聊天表单输入框。
5. 输入 `请用一句话回复 QA 测试消息`。
6. 点击“发送”按钮。
7. 等待助手消息完成生成，或等待出现明确错误。

### 预期结果

- 消息列表出现用户消息 `请用一句话回复 QA 测试消息`。
- 消息列表出现助手消息。
- 生成期间显示加载或思考状态。
- 完成后助手消息状态为正常；失败时显示可理解错误。

### 边缘情况

- 响应较慢时，UI 不应卡死，暂停入口应可用。

### 清理

删除测试数据库中的 `QA 命名助手` 或清空该会话消息。

## 空白输入不能发送

### 测试目标

验证空消息不会创建用户消息或模型请求。

### 数据隔离

用命令行预置空会话 `QA 命名助手`。

### 测试前提

- 当前打开 `QA 命名助手`。

### 测试数据

- 空白内容：三个空格和一个换行。

### 测试步骤

1. 点击聊天表单输入框。
2. 输入三个空格。
3. 按下 `Shift+Enter` 输入换行。
4. 点击“发送”按钮；如果按钮不可用，记录按钮状态。

### 预期结果

- 不新增用户消息。
- 不新增助手消息。
- 发送按钮应保持不可用，或点击后显示输入不能为空的提示。

### 边缘情况

- 清空输入框后，发送按钮状态应恢复为不可用。

### 清理

确认会话没有新增空白消息。

## 模型不可用时发送

### 测试目标

验证模型未配置或不可用时，用户不会误以为消息已正常发送。

### 数据隔离

只修改测试配置，清空测试 API Key 或选择测试环境中不可用模型。

### 测试前提

- 当前打开 `QA 命名助手`。
- OpenAI API Key 已在测试配置中清空，或测试模型 `qa-model-unavailable` 被预置为不可用状态。

### 测试数据

- 消息：`请用一句话回复 QA 测试消息`

### 测试步骤

1. 点击模型选择控件。
2. 在模型列表中点击 `qa-model-unavailable`；如果列表中该模型为禁用态，记录禁用提示文本。
3. 点击聊天表单输入框。
4. 输入 `请用一句话回复 QA 测试消息`。
5. 点击“发送”按钮。

### 预期结果

- 如果模型不可选择，UI 明确提示不可用。
- 如果允许发送，助手消息显示配置缺失或模型不可用错误。
- 不应静默丢失用户输入。

### 边缘情况

- 错误发生后，用户应能更换模型或打开提供方设置恢复。

### 清理

恢复测试配置或删除测试数据目录。
