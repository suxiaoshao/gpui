# 主页 - 对话 - 消息操作测试

## 用例：打开指定消息预览

### 测试目标

验证消息上的预览入口打开的是被点击的那一条消息。

### 数据隔离

测试数据库预置消息，不使用用户真实消息。

### 测试前提

- `QA 命名助手` 中已有用户消息 `请用一句话回复 QA 测试消息`。
- `QA 命名助手` 中已有助手消息 `QA 助手响应内容`。

### 测试数据

- 目标消息：`QA 助手响应内容`。

### 测试步骤

1. 使用测试数据目录启动应用。
2. 在侧边栏点击 `QA 命名助手`。
3. 在消息列表中找到 `QA 助手响应内容`。
4. 点击该消息旁边的预览按钮。
5. 观察消息预览窗口的内容区域。

### 预期结果

- 消息预览窗口打开。
- 内容区域显示 `QA 助手响应内容`。
- 元数据中的会话标题显示 `QA 命名助手` 或等价可识别信息。

### 边缘情况

- 点击用户消息的预览按钮时，应打开用户消息内容，不应沿用上一条助手消息。

### 清理

关闭消息预览窗口，关闭应用并删除测试数据库。

## 用例：复制指定消息

### 测试目标

验证单条消息复制按钮只复制该消息。

### 数据隔离

只复制测试消息，不复制用户真实内容。

### 测试前提

- 当前打开 `QA 命名助手`。
- 消息列表包含 `QA 助手响应内容`。

### 测试数据

- 目标消息：`QA 助手响应内容`。

### 测试步骤

1. 找到内容为 `QA 助手响应内容` 的助手消息。
2. 点击该消息旁边的复制按钮。
3. 在临时文本输入区域粘贴剪贴板内容。

### 预期结果

- 粘贴内容为 `QA 助手响应内容`。
- 不包含相邻用户消息 `请用一句话回复 QA 测试消息`。
- 消息列表没有新增、删除或重排。

### 边缘情况

- 连续复制用户消息和助手消息时，第二次粘贴内容应覆盖为第二条目标消息。

### 清理

清空临时文本输入区域，关闭应用并删除测试数据库。
