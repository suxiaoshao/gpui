# 主页 - 对话 - 生成控制测试

## 暂停生成

### 测试目标

验证生成中的助手消息可以暂停。

### 数据隔离

使用测试会话和测试模型，不发送敏感内容。

### 测试前提

- 当前打开 `QA 长文本会话`。
- 当前模型可用，且能产生较长响应。

### 测试数据

- 消息：`请写三段关于桌面应用测试策略的说明，每段至少三句话。`

### 测试步骤

1. 点击聊天表单输入框。
2. 输入 `请写三段关于桌面应用测试策略的说明，每段至少三句话。`。
3. 点击“发送”按钮。
4. 等助手消息进入加载或思考状态。
5. 点击消息区域或聊天表单中的“暂停生成”按钮。

### 预期结果

- 生成停止。
- 助手消息状态显示为“已暂停”或等价状态。
- 已生成内容保留在消息中。

### 边缘情况

- 对已完成消息点击暂停入口不应改变消息内容；如果入口不存在，也符合预期。

### 清理

清空 `QA 长文本会话` 的测试消息。

## 重发消息

### 测试目标

验证可重发消息会基于当前会话历史重新生成响应。

### 数据隔离

用命令行预置 `QA 命名助手`，其中包含一条用户消息和一条助手消息。

### 测试前提

- 当前打开 `QA 命名助手`。
- 消息列表中有用户消息 `请用一句话回复 QA 测试消息`。

### 测试数据

- 待重发消息：`请用一句话回复 QA 测试消息`

### 测试步骤

1. 找到用户消息 `请用一句话回复 QA 测试消息`。
2. 点击该消息的“重新发送”按钮。
3. 如出现确认对话框，点击“重新生成”或“确认”按钮。
4. 等待新助手响应完成或出现错误。

### 预期结果

- 重发操作有明确反馈。
- 新响应出现在正确位置。
- 原用户消息不被重复创建为多条相同消息，除非产品明确设计为追加新分支。

### 边缘情况

- 模型不可用时，重发应显示错误而不是删除原消息。

### 清理

删除或清空测试会话消息。

## 清空会话取消和确认

### 测试目标

验证清空会话的取消和确认路径。

### 数据隔离

用命令行预置 `QA 命名助手` 及两条测试消息。不要对用户真实会话执行清空。

### 测试前提

- 当前打开 `QA 命名助手`。
- 消息列表至少有两条消息。

### 测试数据

- 会话：`QA 命名助手`

### 测试步骤

1. 点击会话头部的“清空对话”按钮。
2. 在确认对话框点击“取消”。
3. 观察消息列表。
4. 再次点击“清空对话”按钮。
5. 在确认对话框点击“清空”或“确认”按钮。

### 预期结果

- 第 2 步后消息列表保持不变。
- 第 5 步后消息列表为空，并显示空会话状态。
- 清空只影响 `QA 命名助手`，不影响其他 QA 会话。

### 边缘情况

- 空会话中清空按钮应隐藏、禁用或点击后不产生破坏性变化。

### 清理

删除测试会话或保持空会话用于后续测试。
