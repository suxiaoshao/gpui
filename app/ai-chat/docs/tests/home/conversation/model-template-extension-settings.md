# 主页 - 对话 - 模型、模板与扩展设置测试

## 用例：为当前会话选择测试模型

### 测试目标

验证会话级模型选择只影响当前测试会话。

### 数据隔离

使用测试提供方和测试模型配置，不使用真实 API Key 或真实付费模型调用。

### 测试前提

- 测试数据库已预置 `QA 命名助手` 和 `QA 长文本会话`。
- 测试提供方中存在模型 `qa-model-small` 和 `qa-model-large`。

### 测试数据

- 当前会话：`QA 命名助手`。
- 选择模型：`qa-model-small`。

### 测试步骤

1. 使用测试数据目录启动应用。
2. 在侧边栏点击 `QA 命名助手`。
3. 点击会话工具栏中的模型选择控件。
4. 在模型列表中点击 `qa-model-small`。
5. 在侧边栏点击 `QA 长文本会话`。
6. 再次打开模型选择控件。

### 预期结果

- `QA 命名助手` 的模型显示为 `qa-model-small`。
- 切换到 `QA 长文本会话` 后，不应自动继承未保存到该会话的模型选择。
- 不触发真实网络请求。

### 边缘情况

- 如果模型列表为空，模型选择控件应显示未配置或禁用状态，不能允许发送真实请求。

### 清理

关闭应用并删除测试模型配置和测试数据库。

## 用例：套用单提示词模板

### 测试目标

验证选择模板后，模板内容进入当前会话设置或发送上下文。

### 数据隔离

模板 `QA 单提示词模板` 通过命令行预置到测试数据库，不使用用户真实模板。

### 测试前提

- 测试数据库已预置模板 `QA 单提示词模板`。
- 模板内容为 `你是 QA 单提示词助手。`。
- 当前打开 `QA 命名助手`。

### 测试数据

- 模板名称：`QA 单提示词模板`。

### 测试步骤

1. 点击会话工具栏中的模板选择控件。
2. 在模板列表中点击 `QA 单提示词模板`。
3. 打开当前会话的模板或系统提示设置区域。
4. 观察已选模板名称和提示词内容。

### 预期结果

- 当前会话显示已选择 `QA 单提示词模板`。
- 模板内容显示 `你是 QA 单提示词助手。` 或能在发送上下文中被明确引用。
- 其他会话不自动变更模板。

### 边缘情况

- 选择模板列表中的取消或关闭按钮时，当前会话模板保持原值。

### 清理

清除当前会话模板选择，关闭应用并删除测试数据库。

## 用例：关闭扩展设置不保存修改

### 测试目标

验证扩展设置弹窗取消后不会保存临时改动。

### 数据隔离

只修改测试会话 `QA 命名助手` 的测试扩展配置。

### 测试前提

- 当前打开 `QA 命名助手`。
- 扩展设置中存在布尔开关或文本配置项。

### 测试数据

- 文本配置测试值：`QA extension draft value`。

### 测试步骤

1. 点击会话工具栏中的扩展设置按钮。
2. 在扩展设置对话框的文本配置输入框填写 `QA extension draft value`。
3. 点击取消按钮或关闭按钮。
4. 再次点击扩展设置按钮。
5. 观察同一个文本配置输入框。

### 预期结果

- 第 3 步后对话框关闭。
- 第 5 步中输入框不显示 `QA extension draft value`。
- 当前会话没有保存取消前的临时改动。

### 边缘情况

- 如果存在必填字段，取消按钮应始终可关闭对话框，不能因为草稿非法而阻塞取消。

### 清理

关闭扩展设置对话框，关闭应用并删除测试数据库。
