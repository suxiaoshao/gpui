# 主页 - 对话 - 聊天表单测试

## 用例：输入多行消息并保留换行

### 测试目标

验证聊天输入框能输入多行内容，发送前内容不丢失。

### 数据隔离

使用测试会话 `QA 命名助手`，不在用户真实会话中输入测试内容。

### 测试前提

- 测试数据库已预置 `QA 命名助手`。
- 当前没有正在生成的回复。

### 测试数据

- 输入内容：

```text
第一行：QA 多行输入
第二行：请保留换行
```

### 测试步骤

1. 使用测试数据目录启动应用。
2. 在侧边栏点击 `QA 命名助手`。
3. 点击聊天输入框。
4. 在聊天输入框填写 `第一行：QA 多行输入`。
5. 按下换行快捷键。
6. 在第二行填写 `第二行：请保留换行`。
7. 不点击发送，观察输入框内容。

### 预期结果

- 输入框显示两行文本。
- 第一行内容为 `第一行：QA 多行输入`。
- 第二行内容为 `第二行：请保留换行`。
- 未点击发送前消息列表不新增用户消息。

### 边缘情况

- 如果输入框高度增加，工具栏和发送按钮不应遮挡第二行文本。

### 清理

全选并删除输入框内容，关闭应用并删除测试数据库。

## 用例：附件或扩展入口取消后不改变输入

### 测试目标

验证聊天表单中的辅助入口被取消时，不会清空已输入文本。

### 数据隔离

不选择用户真实文件；如弹出文件选择器，只点击取消。

### 测试前提

- 当前打开 `QA 命名助手`。
- 聊天输入框为空。

### 测试数据

- 输入内容：`QA 取消附件后保留文本`。

### 测试步骤

1. 点击聊天输入框。
2. 填写 `QA 取消附件后保留文本`。
3. 点击聊天表单中的附件或扩展入口按钮。
4. 如果出现系统文件选择器，点击取消按钮。
5. 观察聊天输入框。

### 预期结果

- 输入框仍显示 `QA 取消附件后保留文本`。
- 没有新增附件预览。
- 发送按钮状态仍按文本输入有效性展示。

### 边缘情况

- 如果取消后焦点返回输入框，继续输入 ` 追加内容` 时应追加在原文本后方。

### 清理

清空输入框，关闭应用并删除测试数据库。
