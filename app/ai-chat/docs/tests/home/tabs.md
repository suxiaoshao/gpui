# 主页 - 标签页测试

## 用例：打开两个会话标签并切换

### 测试目标

验证主页可打开多个会话标签，并按点击切换当前会话。

### 数据隔离

不要使用用户真实会话。测试前用命令行在测试数据库中预置 `QA 命名助手` 和 `QA 长文本会话`。

### 测试前提

- 应用尚未启动。
- 测试数据库已预置两个会话：`QA 命名助手`、`QA 长文本会话`。

### 测试数据

- 第一个会话：`QA 命名助手`。
- 第二个会话：`QA 长文本会话`。

### 测试步骤

1. 使用测试数据目录启动应用。
2. 在侧边栏点击 `QA 命名助手`。
3. 确认顶部标签栏显示 `QA 命名助手`。
4. 在侧边栏点击 `QA 长文本会话`。
5. 确认顶部标签栏同时显示 `QA 命名助手` 和 `QA 长文本会话`。
6. 点击顶部标签 `QA 命名助手`。

### 预期结果

- 第 3 步打开 `QA 命名助手` 标签。
- 第 5 步打开第二个标签，且不关闭第一个标签。
- 第 6 步后当前会话内容切换回 `QA 命名助手`。

### 边缘情况

- 重复点击侧边栏中的 `QA 命名助手` 时，应聚焦已有标签，不应创建重复标签。

### 清理

关闭应用并删除测试数据库。

## 用例：关闭当前标签

### 测试目标

验证关闭标签只关闭视图，不删除会话数据。

### 数据隔离

测试数据库预置会话，不能对用户真实会话执行关闭或删除操作。

### 测试前提

- 已打开 `QA 命名助手` 和 `QA 长文本会话` 两个标签。
- 当前标签为 `QA 长文本会话`。

### 测试数据

- 待关闭标签：`QA 长文本会话`。

### 测试步骤

1. 在顶部标签 `QA 长文本会话` 上点击关闭按钮。
2. 观察顶部标签栏。
3. 在侧边栏点击 `QA 长文本会话`。

### 预期结果

- 第 1 步后 `QA 长文本会话` 标签从标签栏消失。
- `QA 命名助手` 标签仍显示。
- 第 3 步可重新打开 `QA 长文本会话`，说明关闭标签没有删除会话。

### 边缘情况

- 关闭最后一个标签时，主页应显示空标签状态或默认欢迎状态，不应删除侧边栏会话。

### 清理

关闭应用并删除测试数据库。
