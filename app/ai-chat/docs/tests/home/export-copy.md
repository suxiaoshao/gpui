# 主页 - 导出与复制测试

## 用例：复制当前会话消息

### 测试目标

验证复制操作能复制指定消息内容，并且不修改会话。

### 数据隔离

使用测试会话 `QA 命名助手`。不要复制用户真实消息；不要把真实密钥或隐私内容放入剪贴板。

### 测试前提

- 测试数据库已预置 `QA 命名助手`。
- 会话中存在用户消息 `请用一句话回复 QA 测试消息` 和助手消息 `QA 助手响应内容`。

### 测试数据

- 待复制消息：`QA 助手响应内容`。

### 测试步骤

1. 使用测试数据目录启动应用。
2. 在侧边栏点击 `QA 命名助手`。
3. 在消息列表中找到内容为 `QA 助手响应内容` 的助手消息。
4. 点击该消息旁边的复制按钮。
5. 在不影响用户数据的临时文本输入位置粘贴剪贴板内容。

### 预期结果

- 粘贴结果为 `QA 助手响应内容`。
- 原会话消息数量和内容不变。
- 不出现包含真实用户数据的剪贴板内容。

### 边缘情况

- 如果消息内容为空或复制按钮不可见，应显示禁用态或不执行复制，不能复制上一条消息的内容。

### 清理

清空临时粘贴位置，关闭应用并删除测试数据库。

## 用例：导出会话到测试目录

### 测试目标

验证导出功能把当前会话写入指定测试目录，文件名使用约定前缀。

### 数据隔离

导出目录必须是临时测试目录，不能选择用户桌面、下载目录或真实项目目录。

### 测试前提

- 测试数据库已预置 `QA 命名助手`。
- 已创建空的临时导出目录。

### 测试数据

- 导出文件名前缀：`qa-ai-chat-export`。
- 会话标题：`QA 命名助手`。

### 测试步骤

1. 使用测试数据目录启动应用。
2. 在侧边栏点击 `QA 命名助手`。
3. 点击会话工具栏中的导出按钮。
4. 在文件保存对话框中选择临时导出目录。
5. 在文件名输入框填写 `qa-ai-chat-export-qa-named-assistant.md`。
6. 点击保存按钮。
7. 在临时导出目录中打开导出的文件。

### 预期结果

- 临时导出目录中出现 `qa-ai-chat-export-qa-named-assistant.md`。
- 文件内容包含会话标题 `QA 命名助手` 和消息 `QA 助手响应内容`。
- 应用内会话内容不因导出而改变。

### 边缘情况

- 在文件保存对话框点击取消时，不应创建任何 `qa-ai-chat-export` 文件。

### 清理

删除临时导出目录和测试数据库。
