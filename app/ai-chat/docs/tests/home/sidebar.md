# 主页 - 侧边栏测试

## 新建顶层文件夹

### 测试目标

验证用户可以从侧边栏创建顶层文件夹。

### 数据隔离

使用测试数据库。若侧边栏已有 `QA 根目录`，先在测试数据库中删除该测试文件夹及其子项。

### 测试前提

- app 已使用测试数据目录启动。
- 主窗口已打开，左侧侧边栏可见。

### 测试数据

- 文件夹名称：`QA 根目录`

### 测试步骤

1. 点击侧边栏顶部或操作区的“新建文件夹”按钮。
2. 在“名称”输入框填写 `QA 根目录`。
3. 点击“创建”按钮。

### 预期结果

- 对话框关闭。
- 侧边栏根目录下出现 `QA 根目录`。
- `QA 根目录` 显示为文件夹项，而不是会话项。

### 边缘情况

- 如果“名称”输入框为空，点击“创建”后对话框不应关闭，并应显示必填提示或保持创建不可用。

### 清理

测试结束后删除测试数据库中的 `QA 根目录`。

## 新建子文件夹

### 测试目标

验证用户可以在已有文件夹内创建子文件夹。

### 数据隔离

如果测试目标不是验证“新建顶层文件夹”，先用命令行在测试数据库中预置 `QA 根目录`。

### 测试前提

- 侧边栏根目录下已存在 `QA 根目录`。

### 测试数据

- 父文件夹：`QA 根目录`
- 子文件夹：`QA 子目录`

### 测试步骤

1. 在侧边栏找到 `QA 根目录`。
2. 打开 `QA 根目录` 的上下文菜单。
3. 点击“新建文件夹”。
4. 在“名称”输入框填写 `QA 子目录`。
5. 点击“创建”按钮。
6. 展开 `QA 根目录`。

### 预期结果

- `QA 子目录` 出现在 `QA 根目录` 下。
- 子文件夹不出现在根目录同级位置。

### 边缘情况

- 名称为空时不能创建。
- 与同级已有文件夹重名时，应显示错误或保持原树结构不变。

### 清理

删除测试数据库中的 `QA 根目录` 及子项。

## 编辑文件夹

### 测试目标

验证文件夹名称可编辑，且侧边栏立即刷新。

### 数据隔离

用命令行预置 `QA 根目录/QA 子目录` 后启动 app。

### 测试前提

- 侧边栏中存在 `QA 根目录` 和子项 `QA 子目录`。

### 测试数据

- 原名称：`QA 子目录`
- 新名称：`QA 子目录 - 已编辑`

### 测试步骤

1. 展开 `QA 根目录`。
2. 打开 `QA 子目录` 的上下文菜单。
3. 点击“编辑”。
4. 选中“名称”输入框中的原内容。
5. 输入 `QA 子目录 - 已编辑`。
6. 点击“保存”或“提交”按钮。

### 预期结果

- 对话框关闭。
- 侧边栏中 `QA 子目录` 变为 `QA 子目录 - 已编辑`。
- 原名称不再显示。

### 边缘情况

- 将名称清空并保存时，对话框不应关闭，已有名称不应被清空。

### 清理

删除测试数据库中的 `QA 根目录` 及子项。

## 新建会话

### 测试目标

验证用户可以在指定文件夹中创建会话。

### 数据隔离

用命令行预置 `QA 根目录`，不要在用户真实会话树中创建测试会话。

### 测试前提

- 侧边栏中存在 `QA 根目录`。

### 测试数据

- 名称：`QA 命名助手`
- 图标：`🧩`
- 说明：`生成更好的名字`
- 父文件夹：`QA 根目录`

### 测试步骤

1. 打开 `QA 根目录` 的上下文菜单。
2. 点击“新建会话”。
3. 在“会话名称”或“名称”输入框填写 `QA 命名助手`。
4. 在“图标”输入框填写 `🧩`。
5. 在“说明”输入框填写 `生成更好的名字`。
6. 点击“创建”按钮。
7. 展开 `QA 根目录`。

### 预期结果

- `🧩 QA 命名助手` 出现在 `QA 根目录` 下。
- 点击该会话后，右侧打开对应会话标签页。
- 会话说明可在会话详情区域或搜索结果摘要中被读取。

### 边缘情况

- 漏填名称时不能创建。
- 漏填图标时不能创建。
- 说明为空时，如果产品允许，应创建成功且说明为空；如果产品不允许，应显示明确提示。

### 清理

删除测试数据库中的 `QA 命名助手` 和 `QA 根目录`。

## 编辑会话

### 测试目标

验证会话名称、图标和说明可编辑。

### 数据隔离

用命令行预置 `QA 根目录/QA 命名助手`，包含图标 `🧩` 和说明 `生成更好的名字`。

### 测试前提

- 侧边栏中存在 `QA 命名助手`。

### 测试数据

- 新名称：`QA 命名助手 - 已编辑`
- 新图标：`💡`
- 新说明：`生成更清晰的功能命名`

### 测试步骤

1. 打开 `QA 命名助手` 的上下文菜单。
2. 点击“编辑”。
3. 在“名称”输入框填写 `QA 命名助手 - 已编辑`。
4. 在“图标”输入框填写 `💡`。
5. 在“说明”输入框填写 `生成更清晰的功能命名`。
6. 点击“保存”按钮。

### 预期结果

- 侧边栏显示 `💡 QA 命名助手 - 已编辑`。
- 打开的会话标签页标题同步更新。
- 搜索旧名称不应再命中该会话，搜索新名称应命中。

### 边缘情况

- 漏填名称或图标时不应保存，原会话数据保持不变。

### 清理

删除测试数据库中的该测试会话和文件夹。

## 移动会话

### 测试目标

验证会话可以从根目录移动到文件夹，也可以移回根目录。

### 数据隔离

用命令行预置 `QA 根目录/QA 子目录` 和根目录会话 `QA 命名助手`。

### 测试前提

- `QA 命名助手` 位于根目录。
- `QA 根目录/QA 子目录` 已存在。

### 测试数据

- 会话：`QA 命名助手`
- 目标文件夹：`QA 子目录`

### 测试步骤

1. 在侧边栏拖拽 `QA 命名助手`。
2. 将其放到 `QA 子目录` 上。
3. 展开 `QA 根目录` 和 `QA 子目录`。
4. 再次拖拽 `QA 命名助手`。
5. 将其放回根目录区域。

### 预期结果

- 第一次移动后，`QA 命名助手` 出现在 `QA 子目录` 下。
- 第二次移动后，`QA 命名助手` 出现在根目录。
- 会话消息和说明不丢失。

### 边缘情况

- 拖到原父级同一位置时不应产生重复会话。

### 清理

删除测试数据库中的会话和文件夹。

## 移动文件夹和非法移动

### 测试目标

验证文件夹移动规则，尤其不能把父文件夹移动到自己的后代中。

### 数据隔离

用命令行预置 `QA 根目录/QA 子目录`。

### 测试前提

- `QA 根目录` 下存在 `QA 子目录`。

### 测试数据

- 父文件夹：`QA 根目录`
- 子文件夹：`QA 子目录`

### 测试步骤

1. 拖拽 `QA 根目录`。
2. 尝试将 `QA 根目录` 放到 `QA 子目录` 上。
3. 观察拖放反馈。
4. 放开鼠标。

### 预期结果

- UI 显示不可放置，或放开后树结构保持不变。
- `QA 根目录` 仍在根目录层级。
- `QA 子目录` 仍在 `QA 根目录` 下。

### 边缘情况

- 文件夹移动失败后，侧边栏不应出现重复文件夹或空白项。

### 清理

删除测试数据库中的 `QA 根目录`。

## 删除会话和文件夹

### 测试目标

验证删除操作的取消和确认路径。

### 数据隔离

用命令行预置 `QA 根目录/QA 子目录` 和 `QA 命名助手`。不要删除用户真实会话。

### 测试前提

- 侧边栏中存在 `QA 命名助手` 和 `QA 子目录`。

### 测试数据

- 会话：`QA 命名助手`
- 文件夹：`QA 子目录`

### 测试步骤

1. 打开 `QA 命名助手` 的上下文菜单。
2. 点击“删除”。
3. 在确认对话框点击“取消”。
4. 确认 `QA 命名助手` 仍存在。
5. 再次打开 `QA 命名助手` 的上下文菜单。
6. 点击“删除”。
7. 在确认对话框点击“删除”。
8. 打开 `QA 子目录` 的上下文菜单。
9. 点击“删除”。
10. 在确认对话框点击“删除”。

### 预期结果

- 第一次取消后，会话仍存在。
- 确认删除后，会话消失。
- 确认删除文件夹后，`QA 子目录` 消失。
- 删除确认按钮使用危险操作样式或明确文案。

### 边缘情况

- 删除包含子项的文件夹前，应能让用户明确知道这是破坏性操作。

### 清理

删除剩余测试文件夹和会话。
