# 数据抓取 - 抓取配置

## 功能目标

用户可以配置抓取入口、页码范围和 Cookie，并在配置有效时启动抓取任务。

## 入口

- `数据抓取` 页面。
- `抓取配置` 区域。

## 主要状态

- 未开始：入口链接、起始页、结束页、Cookie 输入框可编辑。
- 正在抓取：所有配置输入框禁用，避免任务运行中变更请求范围。
- Cookie 显示为已填写状态，但主界面不裸露完整 Cookie。
- 页码范围非法时，不启动任务并显示错误。

## 用户可执行动作

- 在 `入口链接` 输入框填写分页入口 URL。
- 在 `起始页` 输入框填写起始页码。
- 在 `结束页` 输入框填写结束页码。
- 在 `Cookie` 输入框填写站点 Cookie。
- 点击 `开始抓取`。

## 重要边界情况

- `起始页` 大于 `结束页` 时，应显示 `起始页不能大于结束页`。
- Cookie 为空时，产品需要明确当前请求是否允许匿名抓取；不能把空 Cookie 当作用户已填写。
- 抓取运行中不能继续编辑 URL、页码和 Cookie。
- 不应在日志、截图或错误弹窗中展示完整真实 Cookie。
