# 数据抓取 - 任务状态与恢复

## 功能目标

用户可以理解当前抓取任务进度，并在中断或失败后选择正确恢复方式。

## 入口

- `数据抓取` 页面。
- `任务状态` 区域。
- `操作区`。

## 主要状态

- `未开始`：展示说明文案，等待用户填写配置。
- `正在抓取`：展示当前页、已完成页数、入库总数和进度条。
- `已中断`：展示下次继续页，并提供 `从中断处继续`。
- `抓取失败`：展示失败页、错误类型、错误详情和停止说明，并提供 `从失败页重试`。
- `抓取成功`：展示完成页数和入库总数。

## 用户可执行动作

- 正在抓取时点击 `中断`。
- 已中断时点击 `从中断处继续`。
- 抓取失败时点击 `从失败页重试`。
- 抓取完成后点击 `去查询` 查看结果。

## 重要边界情况

- 失败后不能静默继续抓取下一页。
- 从失败页重试必须从失败页重新开始，不能跳过失败页。
- 从中断处继续必须从最后成功页的下一页继续。
- 没有中断状态时不应允许 `从中断处继续`。
- 没有失败状态时不应允许 `从失败页重试`。
