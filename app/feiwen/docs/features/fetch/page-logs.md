# 数据抓取 - 页面日志

## 功能目标

页面日志帮助用户观察长时间分页抓取的逐页结果，定位成功、运行中和失败页面。

## 入口

- `数据抓取` 页面底部。
- `页面日志` 表格。

## 主要状态

- 空状态：未开始或没有页面日志。
- 运行中行：状态为 `抓取中`，详情显示正在请求和解析。
- 成功行：状态为 `成功`，展示入库数量和耗时。
- 失败行：状态为 `失败`，用错误样式突出失败页和失败详情。

## 用户可执行动作

- 查看 `页码`、`状态`、`入库`、`耗时`、`详情` 列。
- 根据失败行的页码决定是否点击 `从失败页重试`。
- 在抓取运行中观察最新日志行。

## 重要边界情况

- 最新日志应可见或可通过滚动到达。
- 失败行不能被成功行覆盖。
- 日志详情不能包含完整真实 Cookie。
- 任务重新开始时，日志应能区分新任务和旧任务，避免误判进度。
