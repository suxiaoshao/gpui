# 高级检索 - 条件构建器测试

## 用例：按标题包含查询

### 测试目标

验证文本条件 `标题` + `包含` 能命中预置小说。

### 数据隔离

不要使用用户真实小说库。测试前用命令行向测试数据库预置 `QA 星河测试小说`。

### 测试前提

- app 尚未启动。
- 测试数据库已预置小说 `QA 星河测试小说`。
- 测试数据库已预置另一条不包含 `星河` 的小说，用于确认过滤生效。

### 测试数据

- 字段：`标题`
- 条件：`包含`
- 值：`星河`
- 预期结果：`QA 星河测试小说`

### 测试步骤

1. 使用测试数据目录启动 app。
2. 点击标题栏 `高级检索` 标签。
3. 点击 `添加条件` 按钮。
4. 在新增条件行的字段控件中选择 `标题`。
5. 在条件控件中选择 `包含`。
6. 在值输入框填写 `星河`。
7. 点击标题栏或页面上的 `搜索` 按钮。
8. 观察结果表格。

### 预期结果

- 结果表格包含 `QA 星河测试小说`。
- 不包含标题不匹配的预置小说。
- 标题栏结果摘要显示至少 `1 条结果` 或等价结果数量。

### 边缘情况

- 将值输入框清空后点击 `搜索`，条件行显示 `请输入文本`，结果不应被空值误过滤。

### 清理

关闭 app，删除测试数据库。

## 用例：漏选字段和条件时显示行内错误

### 测试目标

验证字段或条件缺失时不能执行有效查询，并在对应条件行提示错误。

### 数据隔离

使用测试数据库；不修改预置小说数据。

### 测试前提

- app 已打开 `高级检索` 页面。
- 当前条件构建器为空。

### 测试数据

- 空字段状态：`请选择字段`
- 空条件状态：`请选择条件`

### 测试步骤

1. 点击 `添加条件` 按钮。
2. 不选择字段，点击 `搜索` 按钮。
3. 观察新增条件行。
4. 在字段控件中选择 `标题`。
5. 不选择条件，点击 `搜索` 按钮。
6. 观察新增条件行。

### 预期结果

- 第 2 步后条件行显示 `请选择字段` 错误或保持字段必选状态。
- 第 5 步后条件行显示 `请选择条件` 错误或保持条件必选状态。
- 不执行无效查询。
- 结果表格不因无效条件显示错误数据。

### 边缘情况

- 校验失败后已选择的字段 `标题` 不应被清空。

### 清理

点击 `重置` 按钮，关闭 app，删除测试数据库。

## 用例：按数字范围查询字数

### 测试目标

验证数字范围条件能按字数过滤结果。

### 数据隔离

测试数据通过命令行预置，不修改用户真实数据库。

### 测试前提

- 测试数据库存在 `QA 星河测试小说`，字数为 `120000`。
- 测试数据库存在另一条字数为 `30000` 的小说。

### 测试数据

- 字段：`字数`
- 条件：`介于范围`
- 最小值：`100000`
- 最大值：`150000`

### 测试步骤

1. 打开 `高级检索` 页面。
2. 点击 `添加条件` 按钮。
3. 在字段控件中选择 `字数`。
4. 在条件控件中选择 `介于范围`。
5. 在最小值输入框填写 `100000`。
6. 在最大值输入框填写 `150000`。
7. 点击 `搜索` 按钮。
8. 观察结果表格。

### 预期结果

- 结果表格包含 `QA 星河测试小说`。
- 字数为 `30000` 的预置小说不显示。
- 字数列显示 `120000` 或对应格式化值。

### 边缘情况

- 将最大值改为 `90000` 并点击 `搜索`，应显示范围顺序错误或不返回 `QA 星河测试小说`，不能崩溃。

### 清理

点击 `重置`，关闭 app，删除测试数据库。

## 用例：按标签包含全部查询

### 测试目标

验证标签多选条件能命中同时包含多个标签的小说。

### 数据隔离

通过命令行预置标签和小说标签关联，不使用用户真实标签。

### 测试前提

- `QA 星河测试小说` 已关联 `QA 标签玄幻` 和 `QA 标签完结`。
- 另一条预置小说只关联 `QA 标签玄幻`。

### 测试数据

- 字段：`标签`
- 条件：`包含全部`
- 标签：`QA 标签玄幻`、`QA 标签完结`

### 测试步骤

1. 打开 `高级检索` 页面。
2. 点击 `添加条件` 按钮。
3. 在字段控件中选择 `标签`。
4. 在条件控件中选择 `包含全部`。
5. 在标签选择器中搜索 `QA 标签玄幻`。
6. 点击结果 `QA 标签玄幻`。
7. 在标签选择器中搜索 `QA 标签完结`。
8. 点击结果 `QA 标签完结`。
9. 点击 `搜索` 按钮。

### 预期结果

- 结果表格包含 `QA 星河测试小说`。
- 只包含 `QA 标签玄幻` 的预置小说不显示。
- 条件行中显示两个已选标签 chip。

### 边缘情况

- 删除 `QA 标签完结` chip 后重新搜索，结果范围应扩大，不能继续按两个标签过滤。

### 清理

点击 `重置`，关闭 app，删除测试数据库。

## 用例：子组任一满足和排除本组

### 测试目标

验证子组关系和排除开关能组合表达复杂查询。

### 数据隔离

使用测试数据库预置受限和非受限小说。

### 测试前提

- `QA 星河测试小说` 的是否受限为 `否`。
- `QA 受限测试小说` 的是否受限为 `是`。

### 测试数据

- 子组关系：`任一满足`
- 条件一：`标题` `包含` `测试`
- 条件二：`是否受限` `是否` `是`
- 子组排除：开启 `排除本组`

### 测试步骤

1. 打开 `高级检索` 页面。
2. 点击 `添加子组` 按钮。
3. 在子组关系控件中点击 `任一满足`。
4. 在子组中点击 `添加条件`。
5. 第一行字段选择 `标题`。
6. 第一行条件选择 `包含`。
7. 第一行值输入框填写 `测试`。
8. 在子组中再次点击 `添加条件`。
9. 第二行字段选择 `是否受限`。
10. 第二行条件选择 `是否`。
11. 第二行值控件选择 `是`。
12. 开启子组标题处的 `排除本组` 开关。
13. 点击 `搜索` 按钮。

### 预期结果

- 查询不会显示符合子组条件的 `QA 星河测试小说` 和 `QA 受限测试小说`。
- 子组背景或状态能体现 `排除本组` 已开启。
- 应用不因嵌套条件崩溃。

### 边缘情况

- 关闭 `排除本组` 后重新搜索，应能看到符合条件的测试小说。

### 清理

点击 `重置`，关闭 app，删除测试数据库。
