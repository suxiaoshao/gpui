# 高级检索 - 从抓取页进入查询测试

## 用例：抓取成功后进入查询并搜索新数据

### 测试目标

验证抓取成功后，用户能从抓取页进入查询页并搜索到新入库数据。

### 数据隔离

使用 mock HTTP 服务和测试数据库，不访问真实站点。

### 测试前提

- mock 服务返回包含 `QA 星河测试小说` 的测试 HTML。
- 测试数据库为空。
- app 已打开 `数据抓取` 页面。

### 测试数据

- `入口链接`：`http://127.0.0.1:18081/books`
- `起始页`：`1`
- `结束页`：`1`
- `Cookie`：`qa_session=redacted-test-cookie`
- 搜索标题关键词：`星河`

### 测试步骤

1. 在 `入口链接` 输入框填写 `http://127.0.0.1:18081/books`。
2. 在 `起始页` 输入框填写 `1`。
3. 在 `结束页` 输入框填写 `1`。
4. 在 `Cookie` 输入框填写 `qa_session=redacted-test-cookie`。
5. 点击 `开始抓取` 按钮。
6. 等待状态显示 `抓取成功`。
7. 点击 `去查询` 按钮。
8. 点击 `添加条件` 按钮。
9. 字段选择 `标题`。
10. 条件选择 `包含`。
11. 值输入框填写 `星河`。
12. 点击 `搜索` 按钮。

### 预期结果

- 页面切换到 `高级检索`。
- 结果表格显示 `QA 星河测试小说`。
- 不需要重启 app 即可查询到抓取入库数据。

### 边缘情况

- 如果抓取成功但查询无结果，应检查测试数据库路径是否一致；不能改用用户真实数据库继续测试。

### 清理

停止 mock 服务，关闭 app，删除测试数据库。

## 用例：抓取运行中切换到查询页不停止任务

### 测试目标

验证页面切换不会中断运行中的抓取任务。

### 数据隔离

使用延迟 mock HTTP 服务和测试数据库。

### 测试前提

- mock 服务返回多页数据，且第 2 页延迟至少 10 秒。
- app 已打开 `数据抓取` 页面。

### 测试数据

- `入口链接`：`http://127.0.0.1:18081/books`
- `起始页`：`1`
- `结束页`：`3`
- `Cookie`：`qa_session=redacted-test-cookie`

### 测试步骤

1. 填写 `入口链接` 为 `http://127.0.0.1:18081/books`。
2. 填写 `起始页` 为 `1`。
3. 填写 `结束页` 为 `3`。
4. 填写 `Cookie` 为 `qa_session=redacted-test-cookie`。
5. 点击 `开始抓取` 按钮。
6. 等待状态显示 `正在抓取`。
7. 点击 `去查询` 按钮。
8. 等待 3 秒。
9. 点击标题栏 `数据抓取` 标签。
10. 观察任务状态和页面日志。

### 预期结果

- 第 7 步页面切换到 `高级检索`。
- 回到 `数据抓取` 后，任务仍处于 `正在抓取`、`抓取成功` 或明确失败状态。
- 页面日志持续记录运行期间的页码，不因切换页面清空。

### 边缘情况

- 如果运行中发生网络失败，回到抓取页应显示失败页和 `从失败页重试`，不能回到 `未开始`。

### 清理

中断或等待任务结束，停止 mock 服务，关闭 app，删除测试数据库。
