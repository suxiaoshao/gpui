# 高级检索 - 排序规则测试

## 用例：按字数降序排序

### 测试目标

验证排序规则能按字数字段降序排列结果。

### 数据隔离

测试前用命令行预置至少两条 QA 小说，不使用用户真实数据库。

### 测试前提

- `QA 星河测试小说` 字数为 `120000`。
- `QA 短篇测试小说` 字数为 `30000`。
- 当前高级检索没有筛选条件。

### 测试数据

- 排序字段：`字数`
- 排序方向：`降序`

### 测试步骤

1. 打开 `高级检索` 页面。
2. 点击排序区域的添加排序按钮。
3. 在排序字段控件中选择 `字数`。
4. 在排序方向控件中选择 `降序`。
5. 点击 `搜索` 按钮。
6. 观察结果表格前两条 QA 小说。

### 预期结果

- `QA 星河测试小说` 排在 `QA 短篇测试小说` 前面。
- 字数列显示 `120000` 的行在 `30000` 的行之前。

### 边缘情况

- 将方向改为 `升序` 后重新搜索，`QA 短篇测试小说` 应排在前面。

### 清理

点击 `重置`，关闭 app，删除测试数据库。

## 用例：多排序优先级调整

### 测试目标

验证多条排序规则按列表顺序表达优先级，并支持拖拽调整。

### 数据隔离

只使用测试数据库中的 QA 小说。

### 测试前提

- 测试数据库中至少三条 QA 小说，包含相同作者但不同字数的数据。
- 当前高级检索没有排序规则。

### 测试数据

- 排序一：`作者名称` `升序`
- 排序二：`字数` `降序`

### 测试步骤

1. 打开 `高级检索` 页面。
2. 点击添加排序按钮。
3. 在第一条排序字段中选择 `作者名称`。
4. 在第一条排序方向中选择 `升序`。
5. 再次点击添加排序按钮。
6. 在第二条排序字段中选择 `字数`。
7. 在第二条排序方向中选择 `降序`。
8. 点击 `搜索` 按钮。
9. 观察同一作者下的 QA 小说顺序。
10. 拖拽第二条排序规则到第一条排序规则上方。
11. 再次点击 `搜索` 按钮。

### 预期结果

- 第 8 步后先按作者名称分组，再在同一作者内按字数降序。
- 第 10 步后排序列表第一条为 `字数`，第二条为 `作者名称`。
- 第 11 步后结果优先按字数降序排列。

### 边缘情况

- 将排序项拖拽到自身位置时，排序顺序不变。

### 清理

点击 `重置`，关闭 app，删除测试数据库。

## 用例：排序字段为空时阻止查询

### 测试目标

验证排序规则缺少字段时显示错误，不执行无效排序。

### 数据隔离

使用测试数据库，不修改小说数据。

### 测试前提

- app 已打开 `高级检索` 页面。
- 当前没有排序规则。

### 测试数据

- 错误提示：`请选择排序字段`

### 测试步骤

1. 点击添加排序按钮。
2. 如果新排序行默认选择了字段，点击排序字段控件并清空选择；如果无法清空，记录默认字段行为。
3. 点击 `搜索` 按钮。
4. 观察排序行。

### 预期结果

- 如果排序字段可为空，排序行显示 `请选择排序字段`。
- 如果产品默认选择 `标题` 且不允许清空，则不出现空字段错误，查询按 `标题` 默认排序执行。
- 不应出现应用崩溃或无说明失败。

### 边缘情况

- 删除该排序行后点击 `搜索`，查询应按无排序规则执行。

### 清理

点击 `重置`，关闭 app，删除测试数据库。
