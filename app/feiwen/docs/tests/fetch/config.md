# 数据抓取 - 抓取配置测试

## 用例：填写有效配置并开始抓取

### 测试目标

验证抓取配置填写完整且页码范围有效时，可以启动测试抓取任务。

### 数据隔离

只使用本地 mock HTTP 服务 `http://127.0.0.1:18081/books` 和测试数据库。不要访问真实站点，不使用真实 Cookie。

### 测试前提

- app 尚未启动。
- 本地 mock HTTP 服务已启动，并能返回第 1 页和第 2 页测试 HTML。
- 测试数据库为空。

### 测试数据

- `入口链接`：`http://127.0.0.1:18081/books`
- `起始页`：`1`
- `结束页`：`2`
- `Cookie`：`qa_session=redacted-test-cookie`

### 测试步骤

1. 使用测试数据目录启动 app。
2. 点击标题栏 `数据抓取` 标签。
3. 在 `入口链接` 输入框填写 `http://127.0.0.1:18081/books`。
4. 在 `起始页` 输入框填写 `1`。
5. 在 `结束页` 输入框填写 `2`。
6. 在 `Cookie` 输入框填写 `qa_session=redacted-test-cookie`。
7. 点击 `开始抓取` 按钮。
8. 观察 `任务状态` 区域。

### 预期结果

- 抓取任务进入 `正在抓取` 或快速完成为 `抓取成功`。
- 运行期间 `入口链接`、`起始页`、`结束页`、`Cookie` 输入框禁用。
- 页面显示 `Cookie 已隐藏`。
- 日志和状态区域不展示完整 Cookie 值。

### 边缘情况

- 如果 mock 服务不可达，应进入 `抓取失败`，错误类型显示网络错误，不应写入用户真实数据库。

### 清理

停止 mock HTTP 服务，关闭 app，删除测试数据库和测试数据目录。

## 用例：起始页大于结束页时阻止抓取

### 测试目标

验证页码范围非法时不启动抓取任务。

### 数据隔离

使用测试数据目录；不启动真实网络请求。

### 测试前提

- app 已启动并打开 `数据抓取` 页面。
- 当前没有正在运行的抓取任务。

### 测试数据

- `入口链接`：`http://127.0.0.1:18081/books`
- `起始页`：`3`
- `结束页`：`2`
- `Cookie`：`qa_session=redacted-test-cookie`

### 测试步骤

1. 在 `入口链接` 输入框全选并填写 `http://127.0.0.1:18081/books`。
2. 在 `起始页` 输入框全选并填写 `3`。
3. 在 `结束页` 输入框全选并填写 `2`。
4. 在 `Cookie` 输入框全选并填写 `qa_session=redacted-test-cookie`。
5. 点击 `开始抓取` 按钮。
6. 观察 `任务状态` 区域和页面提示。

### 预期结果

- 不进入 `正在抓取`。
- 页面显示 `起始页不能大于结束页`。
- `页面日志` 不新增抓取中、成功或失败行。
- 测试数据库不新增小说记录。

### 边缘情况

- 将 `起始页` 改回 `1` 后再次点击 `开始抓取`，应按有效配置启动任务。

### 清理

清空输入框，关闭 app，删除测试数据目录。

## 用例：运行中配置输入不可编辑

### 测试目标

验证抓取运行中不能修改请求配置。

### 数据隔离

使用延迟响应的本地 mock 服务，确保任务有足够时间处于运行中。

### 测试前提

- mock 服务第 1 页响应延迟至少 5 秒。
- app 已打开 `数据抓取` 页面。

### 测试数据

- `入口链接`：`http://127.0.0.1:18081/books`
- `起始页`：`1`
- `结束页`：`2`
- `Cookie`：`qa_session=redacted-test-cookie`
- 试图修改的新 URL：`http://127.0.0.1:18081/other`

### 测试步骤

1. 在 `入口链接` 输入框填写 `http://127.0.0.1:18081/books`。
2. 在 `起始页` 输入框填写 `1`。
3. 在 `结束页` 输入框填写 `2`。
4. 在 `Cookie` 输入框填写 `qa_session=redacted-test-cookie`。
5. 点击 `开始抓取` 按钮。
6. 在状态显示 `正在抓取` 时点击 `入口链接` 输入框。
7. 尝试全选并填写 `http://127.0.0.1:18081/other`。

### 预期结果

- `入口链接` 输入框处于禁用状态，不能被编辑。
- `起始页`、`结束页`、`Cookie` 输入框也处于禁用状态。
- 抓取任务继续使用启动时的 URL 和页码范围。

### 边缘情况

- 点击 `中断` 后，配置是否恢复可编辑应符合产品设计；但恢复后不能自动改写为用户尝试输入的 `other` URL。

### 清理

中断或等待任务完成，停止 mock 服务，删除测试数据库。
