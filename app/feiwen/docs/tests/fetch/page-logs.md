# 数据抓取 - 页面日志测试

## 用例：成功抓取后显示逐页日志

### 测试目标

验证页面日志记录每页的页码、状态、入库数量、耗时和详情。

### 数据隔离

使用本地 mock 服务和测试数据库。

### 测试前提

- mock 服务第 1 页和第 2 页都返回成功测试 HTML。
- app 已打开 `数据抓取` 页面。

### 测试数据

- `入口链接`：`http://127.0.0.1:18081/books`
- `起始页`：`1`
- `结束页`：`2`
- `Cookie`：`qa_session=redacted-test-cookie`

### 测试步骤

1. 填写 `入口链接` 为 `http://127.0.0.1:18081/books`。
2. 填写 `起始页` 为 `1`。
3. 填写 `结束页` 为 `2`。
4. 填写 `Cookie` 为 `qa_session=redacted-test-cookie`。
5. 点击 `开始抓取` 按钮。
6. 等待状态显示 `抓取成功`。
7. 查看 `页面日志` 表格。

### 预期结果

- 表格显示 `页码`、`状态`、`入库`、`耗时`、`详情` 列。
- 日志中包含页码 `1` 和 `2`。
- 成功页状态显示 `成功`。
- 入库列显示每页入库数量。
- 详情列不包含完整 Cookie。

### 边缘情况

- 如果最新日志在顶部显示，页码 `2` 应位于页码 `1` 上方；如果最新日志在底部显示，顺序应稳定且可理解。

### 清理

关闭 app，停止 mock 服务，删除测试数据库。

## 用例：失败页日志突出显示

### 测试目标

验证抓取失败时，失败页在页面日志中有明确失败状态和详情。

### 数据隔离

使用 mock 服务制造第 2 页失败，不访问真实站点。

### 测试前提

- mock 服务第 1 页成功，第 2 页返回错误或无法解析 HTML。
- app 已打开 `数据抓取` 页面。

### 测试数据

- `入口链接`：`http://127.0.0.1:18081/books`
- `起始页`：`1`
- `结束页`：`2`
- `Cookie`：`qa_session=redacted-test-cookie`

### 测试步骤

1. 填写 `入口链接` 为 `http://127.0.0.1:18081/books`。
2. 填写 `起始页` 为 `1`。
3. 填写 `结束页` 为 `2`。
4. 填写 `Cookie` 为 `qa_session=redacted-test-cookie`。
5. 点击 `开始抓取` 按钮。
6. 等待状态显示 `抓取失败`。
7. 查看 `页面日志` 表格中页码 `2` 的行。

### 预期结果

- 页码 `2` 的状态显示 `失败`。
- 失败行使用错误样式突出。
- 详情列显示失败原因摘要。
- 页码 `1` 的成功日志仍保留。

### 边缘情况

- 失败后不应继续新增页码 `3` 的成功日志。

### 清理

关闭 app，停止 mock 服务，删除测试数据库。
