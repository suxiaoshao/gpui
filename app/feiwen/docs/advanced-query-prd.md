# feiwen 高级检索 PRD

## 背景

当前 feiwen 的查询页只支持一个快速搜索框和标签选择：搜索框匹配标题或作者名称，标签选择要求作品包含所有已选标签。这种方式适合轻量查找，但无法表达更精确的本地书库筛选需求，例如：

- 同时满足多个字段条件，或在多组条件之间使用“任一满足”。
- 排除受限作品、排除特定作者、排除某类标签组合。
- 按字数、阅读数、回复数等数字字段做范围筛选。
- 判断 tags 是否有交集、是否包含一组标签、是否被某组标签覆盖。
- 用多个排序规则控制结果顺序，例如先按“阅读数 / 字数”降序，再按回复数降序。

本功能目标是把 query 页从简单搜索升级为结构化高级检索工具，同时保留当前快速搜索的低门槛体验。

## 目标

- 支持嵌套的布尔条件组：全部满足、任一满足、排除。
- 支持按字段类型展示可用操作符，避免用户选择无效条件。
- 支持文字、数字、布尔、作者、tags 五类字段条件。
- 支持多个排序规则，并用顺序表达优先级。
- 支持排序表达式使用基础数学关系，例如字段相加、相减、相乘、相除。
- 高级检索应能与当前快速搜索和标签选择平滑共存。

## 非目标

- 第一版不设计公开文本 DSL。
- 第一版不要求把查询条件编译成 SQL 或 Diesel 动态表达式。
- 第一版不做查询模板保存、导入、导出。
- 第一版不引入全文搜索引擎或独立索引服务。
- 第一版参考图不要求后续 GPUI 实现逐像素还原。

## 用户流程

1. 用户进入 query 页。
2. 用户可继续使用顶部快速搜索框输入标题或作者关键词。
3. 用户展开“高级检索”区域，新增一个条件组。
4. 条件组默认是“全部满足”，用户可以切换为“任一满足”或“排除”。
5. 用户在条件组中添加条件行，依次选择字段、操作符和值。
6. 用户可以在条件组中继续添加子条件组，实现嵌套。
7. 用户在排序区域添加一个或多个排序规则。
8. 用户点击搜索，列表显示符合条件的作品，并按排序优先级排列。
9. 条件无效时，界面在对应条件行提示错误，不执行不完整查询。

## 查询模型

内部建议使用 typed query model 表达高级检索，而不是直接把 UI 状态散落在视图层。

```text
QuerySpec
  filter: FilterExpr
  sorts: Vec<SortSpec>

FilterExpr
  All(Vec<FilterExpr>)
  Any(Vec<FilterExpr>)
  Not(Box<FilterExpr>)
  Predicate(Predicate)

SortSpec
  expr: SortExpr
  direction: Asc | Desc
  priority: usize
```

`All` 表示全部满足，`Any` 表示任一满足，`Not` 表示排除。UI 中的每个条件组都应能转换为一个 `FilterExpr`。

## 字段与操作符

### 文字字段

适用字段：

- 标题
- 简介
- 最新章节标题
- 作者名称

操作符：

- 包含
- 以什么开头
- 以什么结尾
- 等于

### 数字字段

适用字段：

- 作品 ID
- 最新章节 ID
- 字数
- 阅读数
- 回复数
- 作者 ID

操作符：

- 等于
- 不等于
- 大于
- 大于等于
- 小于
- 小于等于
- 介于范围

缺失数字值默认不命中过滤条件。

### 布尔字段

适用字段：

- 是否受限

操作符：

- 是
- 否

### tags 字段

操作符：

- 与给定标签有交集
- 包含全部给定标签
- 被给定标签集合包含
- 与给定标签集合完全相等
- 为空
- 不为空

### 作者字段

操作符：

- 是某个作者
- 作者在集合中
- 作者不在集合中
- 作者名称包含关键词

作者匹配优先使用作者 ID；没有作者 ID 的匿名作者使用作者名称匹配。

## 排序规则

排序区域支持添加多个排序项，顺序即优先级。每个排序项包含：

- 排序表达式
- 升序或降序
- 启用状态

排序表达式支持：

- 单字段，例如字数、阅读数、回复数、最新章节 ID。
- 基础数学表达式，例如阅读数 / 字数、回复数 + 阅读数。

缺失值、除零、表达式无法计算时，该条记录在该排序项中排在最后。多个排序项应保持稳定排序：低优先级先应用，高优先级后应用。

## UI 要求

- query 页顶部保留快速搜索框和搜索按钮。
- 高级检索区域以结构化条件构建器为主，不使用营销式 hero 或装饰性卡片。
- 条件组需要清晰展示当前关系：全部满足、任一满足、排除。
- 嵌套条件组需要有明显缩进和轻量分隔线，避免层级混乱。
- 条件行使用字段选择、操作符选择和值输入三段式布局。
- tags 和作者集合值应使用可扫描的 token/chip 形式展示。
- 排序规则列表应能看出优先级顺序，并提供升序/降序状态。
- 空状态应引导用户添加第一个条件，而不是展示大量说明文字。

## 错误状态

- 条件行字段为空：提示选择字段。
- 操作符为空：提示选择条件。
- 值为空且该操作符需要值：提示输入值。
- 数字解析失败：提示输入有效数字。
- 范围条件下限大于上限：提示调整范围。
- 排序表达式不完整：提示补全表达式。

错误应出现在对应条件行或排序项附近，不使用全局兜底错误掩盖具体问题。

## 验收标准

- 用户可以创建至少三层嵌套的条件组。
- 用户可以组合全部满足、任一满足、排除三种关系。
- 用户可以对文字、数字、布尔、作者、tags 字段分别选择合法操作符。
- 用户不能为字段选择不适用的操作符。
- 用户可以添加多个排序规则，并明确看到优先级。
- 当前快速搜索和标签选择的原有能力仍可表达。
- PRD 中的参考图能清楚传达高级检索区域、嵌套条件和排序区域的产品方向。
